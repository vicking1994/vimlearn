# 第 0 章：请先读这一章

## 为什么要写这份指南

学习 Vim 的途径很多：`vimtutor` 是很好的起点，`:help` 手册里有你需要的全部参考资料。

然而，普通用户需要的往往比 `vimtutor` 多，又比整本 `:help` 少。本指南试图填补这段空白，只突出关键功能，让你在尽可能短的时间里学会 Vim 中最有用的部分。

你很可能并不需要 Vim 的全部功能。大概只要掌握其中约 20%，就能成为高效的 Vim 用户。本指南会告诉你哪些功能最值得学。

这是一份带有个人倾向的指南。它涵盖的是我在日常使用 Vim 时常用的技巧。各章的顺序，是按照我认为对初学者来说最合乎逻辑的主题来安排的。

本指南以示例为主。学习新概念时，示例不可或缺；足够多的示例能更牢固地巩固这些概念。

有些人可能会问：「我为什么要学 Vimscript？」在用 Vim 的第一年里，我只满足于会用 Vim。随着时间推移，我越来越需要 Vimscript 来写满足自己编辑习惯的自定义命令。当你逐渐精通 Vim 时，迟早会发现需要学 Vimscript。那何不早点开始？Vimscript 是一门很小的语言，在本指南里只用四章就能学到基础。

不懂 Vimscript 也能把 Vim 用得很远，但懂它会帮你走得更远。

本指南面向初学者和进阶 Vim 用户。开头是宽泛而简单的概念，结尾是具体而深入的内容。如果你已经是进阶用户，我仍然建议你从头到尾读一遍，因为你可能会学到新东西！

## 如何从其他文本编辑器过渡到 Vim

学习 Vim 是一件很有成就感的事，尽管并不容易。主要有两种学习方式：

1. 立即全面切换
2. 循序渐进

所谓立即全面切换，是指立刻停用你原来用的编辑器或 IDE，从现在开始只用 Vim。这种做法的缺点是，在第一周或两周里生产力会明显下降。如果你是全职程序员，这种方法可能不现实。所以对大多数人来说，我认为过渡到 Vim 最好的方式是循序渐进地使用它。

要循序渐进地使用 Vim，可以在最初两周里，每天花一小时把 Vim 当作编辑器，其余时间仍可用其他编辑器。许多现代编辑器都带有 Vim 插件。我刚开始时，每天用一小时 VSCode 里流行的 Vim 插件，并逐渐延长使用时间，直到全天都用它。要记住，这些插件只能模拟 Vim 功能的一小部分。要体验 Vim 的完整能力——例如 Vimscript、命令行（Ex）命令以及与外部命令的集成——你需要使用 Vim 本身。

有两个关键时刻让我开始 100% 使用 Vim：一是理解 Vim 具有类似语法的结构（见第 4 章），二是 [fzf.vim](https://github.com/junegunn/fzf.vim) 插件（见第 3 章）。

第一点，当我意识到 Vim 的类语法结构时，我才真正明白那些 Vim 用户在说什么。我不必背几百条互不相关的命令，只需要掌握一小撮命令，就能以很直观的方式组合起来完成很多事。

第二点，能够快速做模糊文件搜索，是我以前在 IDE 里最常用的功能。当我在 Vim 里学会这样做之后，速度大幅提升，从此再也没有回头。

每个人的编程习惯不同。静下心来想一想，你会发现自己总有一两个最爱用的编辑器或 IDE 功能——也许是模糊搜索、跳转到定义，或是快速编译。无论是什么，尽快把它们找出来，并学会在 Vim 里如何实现（Vim 多半也能做到）。你的编辑速度会得到巨大提升。

一旦你能达到原来约 50% 的编辑速度，就该全职转向 Vim 了。

## 如何阅读本指南

这是一份实践导向的指南。要学好 Vim，你需要培养肌肉记忆，而不是只靠脑子记。

你不会靠读一本「如何骑自行车」的指南就学会骑车，你必须真的去骑。

你需要亲自打出本指南里提到的每一条命令。不仅如此，还要重复多次，并尝试不同组合。查一查你刚学会的那条命令还有哪些用法。`:help` 和搜索引擎是你最好的朋友。你的目标不是把一条命令的所有细节都背下来，而是能自然、本能地执行它。

尽管我尽量让本指南按线性顺序展开，有些概念仍不得不提前出现。例如在第 1 章我会提到替换命令（`:s`），尽管它要到第 12 章才详细讲解。为弥补这一点，每当提前提到尚未讲解的概念时，我会给出一个简短的用法说明，而不做深入解释。请多包涵 :)。

## 更多帮助

再给你一个使用帮助手册的技巧：假设你想进一步了解在插入模式下 `Ctrl-P` 做什么。如果只搜 `:h CTRL-P`，你会被带到普通模式下的 `Ctrl-P`，那并不是你要找的插入模式帮助。这时应改搜 `:h i_CTRL-P`。前缀 `i_` 表示插入模式。注意它属于哪种模式。

## 行文约定

大多数与命令或代码相关的短语会用代码样式表示（`像这样`）。

字符串用一对双引号括起来（"像这样"）。

Vim 命令可以缩写。例如，`:join` 可以缩写成 `:j`。通篇指南里，我会混用简写和全称。在本指南中不常出现的命令，我会用全称；经常出现的，我会用简写。不一致之处还请见谅。一般来说，每遇到新命令，都应在 `:help` 里查一下它的缩写形式。

## Vimrc

在指南的若干处，我会提到 vimrc 里的选项。如果你是 Vim 新手，可以把 vimrc 理解成配置文件。

vimrc 要到第 22 章才详细讲。为便于理解，这里简单说明如何设置。

假设你要开启行号选项（`set number`）。如果还没有 vimrc，就创建一个。它通常放在用户主目录，文件名为 `.vimrc`。具体位置因操作系统而异。在 macOS 上，我把它放在 `~/.vimrc`。要确定你该放在哪里，可查看 `:h vimrc`。

在文件里加入 `set number`。保存（`:w`），再执行 `:source %`。此时左侧应显示行号。

如果你不想永久改设置，也可以临时在命令行里执行 `set`，例如运行 `:set number`。缺点是这种设置是临时的，退出 Vim 后就会消失。

既然我们学的是 Vim 而不是 Vi，有一项设置必须有：`nocompatible`。在 vimrc 里加上 `set nocompatible`。当 Vim 以 `compatible` 模式运行时，许多 Vim 专有功能会被关闭。

一般来说，只要文中提到某个 vimrc 选项，就把它写进 vimrc，保存后再 `:source` 即可。
