# 第 2 章：缓冲区、窗口与标签页

如果你用过现代文本编辑器，多半已经熟悉窗口和标签页。Vim 用的是三种显示抽象，而不是两种：
- 缓冲区（buffers）
- 窗口（windows）
- 标签页（tabs）

在本章中，我会说明缓冲区、窗口和标签页分别是什么，以及它们在 Vim 中如何工作。

开始之前，请确保在 vimrc 里设置了 `set hidden`。若没有这项设置，每当你试图从未保存的缓冲区切走时，Vim 都会提示你保存文件（若想快速切换，这会很烦人）。我还没讲过 vimrc 是什么。vimrc 文件通常位于你的主目录，文件名为 `.vimrc`（即完整路径为 `~/.vimrc`）。我的 vimrc 在 `~/.vimrc`。要查看应该把 vimrc 建在哪里，可查阅 `:h vimrc`。若还没有 vimrc，可在终端运行 `vim +w ~/.vimrc` 来创建。在文件中加入：

```
set hidden
```

保存后，再执行 `:source %`（在 vimrc 打开时从 Vim 内运行）。

## 缓冲区

什么是**缓冲区**？

缓冲区是一块内存区域，你可以在其中书写和编辑文本。在 Vim 中打开一个文件时，该文件的数据会绑定到一个缓冲区。在 Vim 中打开 3 个文件，就会为它们各自创建 3 个缓冲区。

请准备两个空文件 `file1.js` 和 `file2.js`（若可以，最好用 Vim 创建，例如在终端运行 `vim +wn file1.js +wq file2.js`）。然后在终端运行：

```bash
vim file1.js
```

你看到的是 `file1.js` 的**缓冲区**。每次在 Vim 中打开新文件，Vim 都会新建一个缓冲区。

退出 Vim。这次一次打开两个新文件：

```bash
vim file1.js file2.js
```

Vim 当前显示的是 `file1.js` 的缓冲区，但它实际上创建了两个缓冲区：`file1.js` 的缓冲区和 `file2.js` 的缓冲区。运行 `:buffers` 可查看所有缓冲区（也可以用 `:ls` 或 `:files`）。列表中应**同时**出现 `file1.js` 和 `file2.js`。执行 `vim file1 file2 file3 ... filen` 会创建 `n` 个缓冲区。每打开一个新文件，Vim 都会为该文件新建一个缓冲区。

在缓冲区之间切换有几种方式：
- `:bnext` 跳到下一个缓冲区（`:bprevious` 跳到上一个）。
- `:buffer` 后跟文件名。按 `<Tab>` 时，Vim 可以为你补全任意文件名。
- `:buffer` 后跟 `n`，其中 `n` 是缓冲区编号。例如，输入 `:buffer 2` 会跳到第 2 号缓冲区。
- 用 `Ctrl-O` 跳到跳转列表中较旧的位置，用 `Ctrl-I` 跳到较新的位置。它们不是专门针对缓冲区的方法，但可以用来在不同缓冲区之间跳转。跳转我会在第 5 章更详细说明。
- 用 `Ctrl-^` 回到最近编辑过的缓冲区。

一旦 Vim 创建了缓冲区，它就会留在缓冲区列表里。要删除当前缓冲区，可输入 `:bdelete`。该命令也可以接受缓冲区编号作为参数（例如 `:bdelete 3` 删除第 3 号缓冲区），或接受文件名（输入 `:bdelete` 后用 `<Tab>` 补全）。

对我来说，学缓冲区最难的是「在脑子里想象」它的工作方式，因为我的思维早已习惯主流编辑器的窗口模型。缓冲区可以比作一副扑克牌：若有 2 个缓冲区，就有叠在一起的 2 张牌。你只能看到最上面那张，但你知道下面还有一张。若屏幕上显示的是 `file1.js` 缓冲区，则 `file1.js` 这张牌在牌堆顶上。这里的另一张牌 `file2.js` 虽然看不见，但它确实存在。若切换到 `file2.js` 缓冲区，则 `file2.js` 到了牌顶，`file1.js` 在下面。

若你以前没用过 Vim，这是新概念。请慢慢消化。

## 退出 Vim

顺便一提，若打开了多个缓冲区，可以用「全部退出」一次关掉：

```
:qall
```

若要不保存修改就全部关闭，在末尾加上 `!`：

```
:qall!
```

要全部保存并退出，运行：

```
:wqall
```

## 窗口

窗口是**观察**缓冲区的一块视口。若你来自主流编辑器，这个概念可能并不陌生。大多数文本编辑器都能显示多个窗口。在 Vim 里同样可以有多窗口。

我们再次从终端打开 `file1.js`：

```bash
vim file1.js
```

前面我说你看到的是 `file1.js` 缓冲区，这话没错，但不完整。你看到的是 `file1.js` 缓冲区，但它是通过**一个窗口**显示出来的。窗口就是你用来「看」缓冲区的界面。

先别退出 Vim。运行：

```
:split file2.js
```

现在你通过**两个窗口**在看两个缓冲区。上方窗口显示 `file2.js` 缓冲区，下方窗口显示 `file1.js` 缓冲区。

若要在窗口之间移动光标，可用这些快捷键：

```
Ctrl-W h    光标移到左侧窗口
Ctrl-W j    光标移到下方窗口
Ctrl-W k    光标移到上方窗口
Ctrl-W l    光标移到右侧窗口
```

接着运行：

```
:vsplit file3.js
```

现在你看到三个窗口、三个缓冲区：一个窗口显示 `file3.js` 缓冲区，另一个显示 `file2.js` 缓冲区，还有一个显示 `file1.js` 缓冲区。

多个窗口可以显示**同一个**缓冲区。在左上方的窗口中输入：

```
:buffer file2.js
```

现在上方两个窗口都在显示 `file2.js` 缓冲区。若在任一显示 `file2.js` 的窗口里输入文字，两个含有该缓冲区的窗口都会实时更新。

要关闭当前窗口，可运行 `Ctrl-W C` 或输入 `:quit`。关闭窗口后，缓冲区仍然存在（运行 `:buffers` 可以确认）。

下面是一些有用的普通模式窗口命令：

```
Ctrl-W V    新建垂直分割
Ctrl-W S    新建水平分割
Ctrl-W C    关闭窗口
Ctrl-W O    只保留当前窗口，关闭其余窗口
```

以及若干有用的窗口相关命令行命令：

```
:vsplit filename    垂直分割窗口
:split filename     水平分割窗口
:new filename       新建窗口
```

请慢慢熟悉它们。更多信息可查阅 `:h window`。

## 标签页

标签页是**一组**窗口的集合。可以把它想成窗口的一种布局。在大多数现代文本编辑器（以及现代浏览器）里，一个标签通常对应一个打开的文件或页面；关掉标签，那个文件或页面也就没了。在 Vim 里，标签页**不代表**打开的文件。在 Vim 中关闭一个标签页，并不是在关闭文件；你只是关掉了那种窗口布局。在该布局里打开的文件并没有被关掉，它们仍在各自的缓冲区里。

下面看 Vim 标签页的实际用法。打开 `file1.js`：

```bash
vim file1.js
```

要在新标签页中打开 `file2.js`：

```
:tabnew file2.js
```

你也可以按 `<Tab>` 让 Vim 在**新标签页**里补全要打开的文件名（这里可不是在玩「Tab」的双关）。

下面是一些有用的标签页导航命令：

```
:tabnew file.txt    在新标签页中打开 file.txt
:tabclose           关闭当前标签页
:tabnext            跳到下一个标签页
:tabprevious        跳到上一个标签页
:tablast            跳到最后一个标签页
:tabfirst           跳到第一个标签页
```

也可以用 `gt` 跳到下一个标签页（用 `gT` 可跳到上一个）。可以给 `gt` 加数字前缀，表示标签序号；例如，要跳到第三个标签页，输入 `3gt`。

使用多个标签页的一个好处是：不同标签页里可以有不同的窗口排布。也许你想第一个标签页里是 3 个垂直窗口，第二个标签页里则是横竖混合的布局。标签页非常适合做这种事。

要从终端启动带多个标签页的 Vim，可以运行：

```bash
vim -p file1.js file2.js file3.js
```

## 在「三维」中移动

在窗口之间移动，就像在笛卡尔坐标系的 X–Y 平面上二维移动。你可以分别用 `Ctrl-W K/L/J/H` 移到上、右、下、左方的窗口。

在缓冲区之间移动，则像在 Z 轴上穿行。想象你的缓冲区文件沿 Z 轴排开。你可以用 `:bnext` 和 `:bprevious` 一次沿 Z 轴移动一个缓冲区；也可以用 `:buffer 文件名/缓冲区号` 跳到 Z 轴上的任意位置。

把窗口移动与缓冲区移动结合起来，就可以在**三维空间**里移动：用窗口命令在上、右、下、左（X–Y 导航）之间切换；由于每个窗口里都有一组缓冲区，你还可以用缓冲区命令向前、向后（Z 导航）移动。

## 更聪明地使用缓冲区、窗口与标签页

你已经了解了缓冲区、窗口和标签页是什么，以及它们在 Vim 中如何工作。理解得更清楚之后，就可以按自己的工作流来使用它们。

每个人的工作流都不同。下面是我的做法，仅供参考：
- 首先，我用缓冲区存放当前任务所需的全部文件。在变慢之前，Vim 可以打开很多缓冲区。而且开很多缓冲区不会让屏幕显得拥挤。任何时候我只「看到」一个缓冲区（假设只有一个窗口），这样更容易专注。需要跳转时，我可以随时快速飞到任意已打开的缓冲区。
- 我用多个窗口同时查看多个缓冲区，常见场景是对比文件差异、阅读文档或顺着代码调用链看。我尽量把窗口数量控制在不超过三个，否则小笔记本的屏幕会很挤。用完后我会关掉多余的窗口。窗口越少，干扰越少。
- 我不怎么用 Vim 的标签页，而是用 [tmux](https://github.com/tmux/tmux/wiki) 的窗口。我通常会同时开多个 tmux 窗口，例如一个窗口看前端代码，另一个看后端代码。

我的工作流可能和你的编辑习惯不同，这没关系。多试验，找到适合自己的节奏和编码风格。
